最近接到一个需求，要求根据用户的 6 个项目的完成状态，对应展示 1，2，3，4，5，6 正多边形，而且后续要动画展示各条边，引导到各个角上。

  • 根据状态展示不同的正多边形
  • 单线是渐变色，旁边有虚化的单色阴影
  • 单线绘制动效

大概效果如图：

这时想了一下有啥可以比较快实现了？

  • css 可以绘制实心的三角形(实心？不行，过)
  • canvas 可以
  • svg 可以

算多边形点位置

无论是用 canvas 还是 svg，我们还是要知道绘制的点的位置的，所以我们第一步就是要求各个点的位置(我是不会告诉你，这篇文章的主要目的是记住这个计算的…)

因为这里用的都是正 n 边形，根据角度和半径就能算出各个点的位置

绘制多边形

svg 画法

绘制多边形

svg 中有 polygon 这个标签绘制多边形

  • width/height 定义 svg 的可视区域
  • viewBox=”x, y, width, height” // x:左上角横坐标，y:左上角纵坐标，width:宽度，height:高度 （将 viewBox 的区域铺满整个 svg）

绘制多边形的渐变单线

  1. 定义渐变元素
  • gradientUnits 属性定义了渐变的范围，默认值是objectBoundingBox,表示对象的外包矩形，所以垂直或者水平的渐变线是画不出来的（因为垂直的线的 width 是 0，导致区域不是一个面了）；userSpaceOnUse不定义x1、y1、x2、y2的话，默认是整个 svg 大小；

如下图中，垂直的线的外包矩形的width是 0，导致渐变线无法绘制，而非垂直线则可以

下图中，左图是gradientUnits="userSpaceOnUse",它能绘制出垂直的渐变线段，但这里的渐变色范围是整个svg而不是单条线段（导致每条线段的渐变都不一致）；
右图是gradientUnits="objectBoundingBox"，它绘制出来的渐变线段是一致的，但垂直的线段无法绘制。

canvas 画法

绘制多边形

  • 根据canvas的宽高大小，计算绘制的半径，从而得到需要绘制的正多边形的顶点位置
  • 根据计算得到的顶点位置，绘制路径
  • 设置线的颜色，线宽

绘制多边形的渐变单线

  • 定义渐变元素
  • 利用阴影实现底线虚化
  • 绘制线段

单线动画

canvas 的动画主要是使用了requestAnimationFrame函数，每帧绘制。这里的Animate是自行封装的一个动画类，给出起始状态、结束状态、持续时间，在onTick函数中计算出中间状态，再自行处理。

优缺点比较

  • 绘制出多边形：svg 绘制的多边形能随父元素的大小的变化而变化（不需要改动 svg 代码），但使用 canvas 绘制的话，改动父元素的大小时，需自行再计算一次点位置并重新绘制图
  • 绘制渐变色的线段：svg 中，如果linearGradient的属性gradientUnits为 "objectBoundingBox"时，垂直或者水平的渐变线段绘制不出来，使用gradientUnits="userSpaceOnUse"的话，是相当于整个 svg 的渐变范围（和目标有点差距）；canvas 能符合要求的绘制出渐变线段
  • 线段绘制动画：svg 能利用 css3 动画实现绘制动画，而 canvas 需要使用 js 的requestAnimationFrame函数，逐帧计计算绘制。
