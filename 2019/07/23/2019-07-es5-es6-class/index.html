ES5 & ES6 的类

Jul 23, 2019 | 阅读

希望能通过对比ES5和ES6的类来更好的理解javascript的类

ES6 的class可以看作只是一个语法糖，它的绝大部分功能，ES5 都可以做到，但有些细节还是有些差异的

定义

ES5 版

ES6版

  • ES6 class的数据类型就是函数，类本身就指向构造函数 (同ES5)

  • ES6 class的所有方法都定义在类的prototype属性上面

  • ES6 class的内部所有定义的方法，都是不可枚举的(包括静态方法) (与ES5不同)

  • ES6 class声明不会提升，但存在暂时性死区。即ES6类的使用要在定义之后（与ES5不同）

  • ES6 class 声明内部默认就是严格模式（ES5需使用use strict指定运行模式）
  • ES6class内部只能定义方法，不能定义属性(与ES5不同)
  • 必须使用 new 调用 ES6 class (与ES5不同)

继承

ES5

ES6

  • ES6的继承是先通过父类的构造函数完成后才得到子类的this（ES5的是先创造子类的实例对象this，然后再将父类的方法添加到this上面）

    • ES6的继承中，子类必须在constructor方法中调用super方法
    • ES6的继承中，在子类的构造函数中，只有调用super之后，才可以使用this关键字，否则会报错

  • ES6中，父类的静态方法，也会被子类继承
