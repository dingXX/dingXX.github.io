之前有写了koa2的简单使用，但实际开发中，怎么会无bug了，调试是开发中必不可少的一个环节，前端开发时通常是在chrome的开发者工具中调试代码的，但nodeJs作为后端代码又可以怎么调试了。一开始我用的是console.log，但之后发现效率极低，所以就整理了一下常用的node调试的方法，有chrome的开发者工具调试(和前端调试方式一致)，有vscode自带的调试（类似于IDE的调试）

Chrome 浏览器的”开发者工具”调试

Chrome DevTools 55+

步骤1 启动开发者工具

步骤2 打开chrome-devtools窗口调试

方法1 在chrome中打开chrome://inspect/#devices页，回车后看到以下内容，点击Open dedicated DevTools for Node按钮，会弹出chrome-devtools窗口

方法2
在chrome中打开要调试的链接，如http://127.0.0.1:3000/user/index，打开开发者工具，选择开发者工具里的左上角的node的图标，就可以看到node的devtools窗口

我们在devtools窗口的Node选项里，能看到我们的后端代码，但这里的代码可能只包含了当前运行的逻辑引用的文件。如果想要引入全部代码，可以切换到Filesystem,添加目录到工作项目，再断点调试（一定要在node对应的开发者工具中断点，在普通的开发者工具里断点是无效的）

多个项目调试

启动调试时，可以设置调试端口号,如--inspect=5858（默认调试端口号为9229）

只是这么设置后，在chrome://inspect/#devices是监测不到页面的，我们需要将设置的调试端口号(5858)给添加到chrome配置里

点击Discover network targets的config按钮，将目标地址添加进来。

为不同的项目，定义不同的监听端口号，就能同时调试多个项目啦。

Visual Studio Code 调试

Visual Studio Code具有对Node.js运行时的内置调试支持

  1. 在vscode中打开launch.json文件（调试–>打开配置），设置调试配置信息
  2. 启动调试（F5 或调试–>启动调试）
  3. 切换断点(F9 或 调试–>切换调试)

具体的vscode调试详情可看vscode 调试
