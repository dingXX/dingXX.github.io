优化

如果将所有的路由都写在了index.js的话，那每新增一个路由，就需要在index.js中定义一个路由，那最后index.js一定异常的长，所以我们需要将路由的定义抽取到一个文件或多个文件中，然后在index.js中导入这些路由处理函数，这样代码就分离了，逻辑也清晰了。

而我们实际开发中，通常是以一个一个功能为单位进行开发的，即一个项目中包含多个功能，一个功能中包含多个路由处理。如下这里的结构：

所以我们可以通过编写一个遍历读取controllers文件夹下的*Controller.js文件中的路由定义的函数

  • 路由的定义规则
    • 如果是页面路由，则为 controllers名 + 具体路由名 如 /users/about
    • 如果是api路由，则为’api/‘ + controllers名 + 具体路由名, 如/api/user/get-user-info
    • / 和/index的路由表示是一致的，只需要定义/index的配置就好

请求数据获取

我们有时需要根据客户端传回的参数来做相关的处理，所以需要在koa中获取请求参数数据

get请求参数

在koa中，get请求的参数，可以直接通过上下文直接获取或通过上下文的request对象获取

  • ctx.query 返回请求对象 如 { a: '1', b: '2' }
  • ctx.querystring 返回请求对象字符串 如a=1&b=2
  • ctx.request.query 返回请求对象 如 { a: '1', b: '2' }
  • ctx.request.querystring 返回请求对象字符串 如a=1&b=2

post请求参数

在koa中，post请求的参数没有提供方法获取，需要自行通过解析上下文中的原生node.js对象requset(为koa上下文的req)来获取
