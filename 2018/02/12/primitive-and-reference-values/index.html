一个变量可以存放两种类型的值，基本类型的值（primitive values）和引用类型的值（reference values）。

区别

存储位置

基本类型：存储在栈（stack）中的简单数据段，也就是说，它们的值直接存储在变量访问的位置。

引用类型：存储在堆（heap）中的对象，也就是说，存储在变量处的值是一个指针（point），指向存储对象的内存处。

引用值类型还在栈中存储了指针，该指针指向堆中该实体的起始地址。当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后从堆中获得实体。

原因

栈空间小，但可以被解释器直接检索，响应速度快；堆空间大，但不能被程序栈直接检索，响应速度稍慢。

基本类型

在 JavaScript 中有5种基本类型（也称为简单数据类型）： String、Number、Boolean、Undefined和Null，(string类型在其他语言中存储在堆中)

比较

原始值的比较是值的比较：只有在它们的值相等时它们才相等。同样，对于字符串来说则并不明显：如果比较两个单独的字符串，当且仅当他们的长度相等且每个索引的字符都相等时，JavaScript 才认为它们相等。

赋值

在从一个变量向另一个变量赋值原始值类型时，会在该变量上创建一个新值，然后再把该值复制到为新变量分配的位置上

原始值类型在赋值操作后，两个变量是相互不受影响的。

引用类型

常用的引用类型有：Object、Array、Date、Function和RegExp

比较

引用值的比较是对象地址的比较，只有两个变量的对象地址指向的是同一个对象时，才算相等。

赋值

当从一个变量向另一个变量赋值引用类型的值时，同样也会在栈中将存储在变量中的对象的值（其实是对象的地址）复制一份放到为新变量分配的空间中，但实际上两个变量指向同一个对象

引用类型的赋值其实是对象保存在栈区地址指针的赋值，因此两个变量指向同一个对象，任何的操作都会相互影响

大家都知道赋值有 按值传递、按引用传递这两种类型，其实ECMAScript中还有一种 叫 按共享传递；

运用理解

函数参数传递

在JavaScript中无论是基本类型还是引用类型，函数参数都是按值传递的，基本类型和引用类型的区别在于传进函数的是什么(基本类型传的是对应的值，引用类型传的是对象的地址)
