this是JavaScript中的一个关键字，但this在JavaScript中与在其他语言中的含义不一样（java中的this的含义是明确具体的，但JavaScript中的this会根据实际情况而变），同时在严格模式(strict mode)与非严格模式下也是有区别的。

简单理解就是，JavaScript中对this的理解，它指代一个对象（object），也就是那个上下文中的主语，或者说运行时的主体。因此重点是要找到运行时的上下文，那this就指向这对象。而在严格模式下，在函数作用域下，当this指向的是全局变量（浏览器中的window）时，this会为undefined。

##全局作用域中的this

在全局作用域下（即this不在任何函数中调用），this指向的是全局变量。

##函数作用域中的this

在函数作用域下调用this，this的值会根据函数的调用方式不同而不同（关键还是确定调用函数的对象）
不同函数调用方法下的this

###直接调用

当代码在浏览器中执行的时候，所有的全局变量和函数都被定义在 window对象上，因此，当我们在全局函数中使用 this 的时候，它会指向全局 window 对象并且拥有它的值（除非在严格模式下）。

###作为对象的方法调用

当以对象里的方法的方式调用函数时，它们的 this 指向调用该函数的对象

调用f函数的对象是o，因此this指向的是o。

在这个例子中，函数f是由o.b对象调用的，因此它的this指向的是o.b。

###在原型链中调用

如果该方法存在于一个对象的原型链上，那么this指向的是调用这个方法的对象，表现得好像是这个方法就存在于这个对象上一样。

因为f函数在p对象的原型链上，所以它的this指向了p

###作为构造函数调用

当一个函数被作为一个构造函数来使用（使用new关键字），它的this与即将被创建的新对象绑定。

在构造函数内部,如果被调用的函数没有显式的 return 表达式，则隐式的会返回 this对象,也就是新创建的对象。

但当在构造函数中，显式 return 值，情况就不一样了。

当显式的 return 表达式后将会影响返回结果，但仅限于返回的是一个对象。(当构造函数显式返回非对象类型值时，系统会忽略显示的return值，而返回this)

###嵌套函数作用域中的this

闭包不能直接通过使用 this 来访问外层函数的 this 变量，因为 this 变量只有当前函数本身可以访问，而其内层函数是访问不到的

在匿名函数内部的 this 不能获得外层函数 this 的值，所以当没有使用严格模式的时候，它就被绑定在了全局 window 对象上了。

###DOM事件处理函数的调用

当函数被用作事件处理函数时，它的this指向触发事件的元素

###通过call或apply或bind方法调用

当一个函数的函数体中使用了this关键字时，通过所有函数都从Function对象的原型中继承的call()方法和apply()方法调用时，它的值可以绑定到一个指定的对象上。

使用 call 和 apply 函数的时候要注意，如果传递的 this 值不是一个对象，JavaScript 将会尝试使用内部 ToObject 操作将其转换为对象。

JavaScript会对call的第一个参数（7）做new Number(7) 的操作，使之转换为一个对象Number {[[PrimitiveValue]]: 7}。

调用f.bind(someObject)会创建一个与f具有相同函数体和作用域的函数，但是在这个新函数中，this将永久地被绑定到了bind的第一个参数，无论这个函数是如何被调用的。
