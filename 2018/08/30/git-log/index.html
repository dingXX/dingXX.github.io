git log简单使用

git log主要是用于查看历史提交

通过直接在git项目的命令行中输入git log，能查看到当前分支的历史提交记录
这是按提交时间排序的，最近的更新排在最上面，列出每个提交的相关信息（如 SHA-1校验和、作者名称、邮箱、时间、提交说明）

git log 的参数

当然，上面这种是不带参数的情况，实际上git log命令有许多常用的参数选项，以便更容易满足大家的需求

控制git log数据展示的的参数

参数 作用 例子
-p 显示每次提交的内容差异 git log -p
--stat 显示每次提交的简略统计信息 git log --stat
--pretty= 不同的形式显示log git log --pretty=short (oneline/full/fuller/format:”%h - %an,%ar:%s”)
--graph 图形展示分支、合并历史 git log --graph
--abbrev-commit 只显示SHA-1的前几个字符 git log --abbrev-commit
--relative-date 时间信息用相对时间 git log --relative-date 时间展示为：2 months ago
--name-only 展示该提交中被提交的文件清单 git log --name-only a.txt
--name-status 展示该提交中被提交的文件清单加对应的操作 git log --name-status M a.txt

git log --pretty=format中的format的常用选项

选项 说明 其他
%H 提交对象（commit）的完整的SHA-1 %h简短SHA-1
%T 树对象（tree）的完整SHA-1 %t简短SHA-1
%P% 父对象（parent）的完整SHA-1 %p简短SHA-1
%an 作者名字 %cn为提交者名字
%ae 作者邮箱 ce为提交者邮箱
%ad 作者修订时间 %ar按多久以前的方式显示
%cd 提交日期 %cr按多久以前的方式显示
%s 提交说明

作者：实际作出修改的人； 提交者将工作成果提交到仓库的人

栗子

控制git log输出的参数

参数 作用 例子
-(n) 仅展示最近的n条数据 git log -10 只展示最近的10条数据
--since、--after 显示指定时间之后的提交 git log --since=2weeks两周内的提交记录
--until、--before 显示指定时间之前的提交 git log --until=2weeks两周内的提交记录
--author 指定作者 git log --author=cynthia
--committer 指定提交者相关 git log --committer
--grep 带关键字查(只是对提交信息内容的关键字) git log --grep=keywork
-Sfunciton_name 列出那些添加或移除了某些字符串的提交(没有空格) git log -S改动 提交的具体内容中包含“改动”两字的提交历史
--merge 显示任何一边接触了合并冲突文件的提交（在冲突时可用） git log --oneline --left-right --merge

git log的高级使用

文件级别的提交历史查询

参数 作用 例子
git log pathName 某些文件或者目录的历史提交 git log a.txt 查看a.txt的提交历史
git log -L :functionname:fileName 查看filename中的fuctionname的历史记录 git log -L :改动:a.txt

分支间的提交历史查询

参数 作用 例子
git log BranchA 查看A分支的提交历史
git log BranchA BranchB 查看A分支或者B分支的提交（取并集）
git log ^BranchA BranchB A分支没有，B分支有的
git log BranchA BranchB --not BranchC 被A分支或B分支包含，但不被C分支包含的提交
git log BranchA..BranchB 查看B分支有，但A分支没有的提交 git log BranchA..BranchB == git log ^BranchA BranchB
git log BranchA...BranchB A分支或B分支有，但不是两者共有的提交 --left-right可显示每个提交属于哪一侧分支
