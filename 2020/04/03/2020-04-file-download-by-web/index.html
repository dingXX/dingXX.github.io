web前端下载文件

Apr 3, 2020 | 技术 | 阅读

之前需要直接根据前端的用户操作内容，由前端生成并下载 文件，所以大概搜了一下实现方法，这里记录一下~

利用a标签下载

a标签 利用 download属性，实现下载，下载的内容为href属性的内容 [不添加download属性的话，只是打开新标签查看]

确定文件路径的下载

直接将文件路径放到href属性中，实现下载 （跨域文件不支持，详情看注意事项）

利用Blob对象生成URL对象 创建文件下载

目标： 生成一个URL对象，将其放到a.href里，实现文件的创建

方法

  • 创建Blob对象

    Blob 对象表示一个不可变、原始数据的类文件对象。Blob 表示的不一定是JavaScript原生格式的数据。File 接口基于Blob，继承了 blob 的功能并将其扩展使其支持用户系统上的文件

  • 创建URL对象

自动下载

利用js 动态创建 a标签，填充要下载的内容，模拟点击下载

注意事项

  1. 当要下载的文件是跨域文件时 ，会下载失败，变成打开查看模式 （即使文件设了Access-Control-Allow-Origin:*，还是查看状态）
    解决方案
  • 如果是设置了Access-Control-Allow-Origin:*的图片，可以利用canvas的toDataURL方法将图片转为URL对象 ，从而绕过跨域限制
  • 利用Fetch API 获取文件的Bolb对象 （文件也要有Access-Control-Allow-Origin:*）
