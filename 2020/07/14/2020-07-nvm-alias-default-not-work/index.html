nvm alias default 不起效问题解决

Jul 14, 2020 | 阅读

nvm alias default <version>按nvm的文档说明，是个可以设置shell中的默认node 版本的

但在我的mac电脑上却一直都没有起效（无论我将nvm alias default设为哪个版本，每次新开一个shell，再node -v都是v.8.12.0），后来开始查原因，终于找到解决办法，记录一下

背景

我的电脑应该是先全局装了个node7(忘了怎么装的呢，貌似不是用brew装的)，再安装的nvm

解决

网上查这个问题时，发现有人提到了~/.bash_profile文件是否有定义node path，所以我就看了一下自己的~/.bash_profile文件，发现确实在path中定义了node 的路径

然后我就尝试将path中的node路径删除，再重启~/.bash_profile文件

再新开一个shell 后，node -v得到的是7.10.1(我配置的是nvm alias default 12)，所以nvm alias default还是没起效

用which node看了一下node的路径，发现是/usr/local/bin/node，并不是nvm安装的node，所以在想，是否node的全局查找变成了我之前安装的node7版本了

然后，我又将自己全局安装的node7版本给删了

再开个shell，这时报了command not found: node的错，只有在这个shell中执行nvm use后，才可以使用node

再google ，有提到了~/.bash_profile中的 export PATH 和 export NVM_DIR 的顺序问题，我尝试调整一下.bash_profile中的export PATH 和 export NVM_DIR顺序（export PATH在前，export NVM_DIR后定义），再重新启动一下bash_profile，这时nvm alias default终于起效了！

/usr/local/opt/nvm/nvm.sh中会修改path，如果自定义的path在nvm.sh之后执行，那nvm.sh设置的path 就无效了，导致nvm alias default无法起效

总结

遇到nvm alias default <version>不起效时，先检查一下是否存在以下问题

  1. 在~/.bash_profile自定义了path路径
  2. ~/.bash_profile中的nvm相关配置定义在path定义之前

处理方法：
保证nvm/nvm.sh的执行在export path之后
